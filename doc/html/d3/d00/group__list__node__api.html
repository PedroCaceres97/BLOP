Detailed Description

Node allocation, cloning, destruction, and field access.

Warning
Thread Safety
Node operations are not thread-safe if they interact with a list. Standalone nodes are safe as long as user code prevents data races.

Function Documentation

◆ blopfn_node_create()

blopt_node blopfn_node_create ( )

Allocate a new standalone node (not linked to any list).

Ownership:
Caller owns the node and must destroy it with blopfn_node_destroy or by providing deallocate == true into any list erasing operation.

Complexity: O(1)

Returns
Newly allocated node, or NULL.

◆ blopfn_node_destroy()

void blopfn_node_destroy ( blopt_node node)

Destroy a node not currently linked to any list.

Ownership rules:

  • Frees the node object.
  • If BLOP_LIST_DEALLOCATE_DATA is defined then BLOP_LIST_DEALLOCATE_DATA(node->data) will be called to deallocate data.

Complexity: O(1)

Parameters
nodeNode to destroy.
Warning
Precondition: The list must be empty (size == 0).
